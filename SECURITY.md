# Security Policy

AI+SU runs inside SketchUp and can execute Ruby code generated by an AI model. That makes security review part of normal use, not an afterthought.

## Supported Versions

Security fixes target the latest release and the current `main` branch.

| Version | Supported |
| --- | --- |
| 1.x | Yes |

## API Keys

Do not commit real DeepSeek, Gemini, Qwen, Kimi, OpenAI-compatible, or image-generation API keys.

The plugin stores keys through SketchUp local settings via `Sketchup.write_default`. Keys should remain on the user's machine and should not appear in source files, issue reports, screenshots, release assets, or Git history.

If a key is exposed:

1. Revoke or rotate it immediately in the provider dashboard.
2. Remove it from local files.
3. Clean the Git history if it was committed.
4. Avoid posting the leaked value in public GitHub discussions.

## Reporting a Vulnerability

Please do not open a public issue with exploit details, private model files, customer drawings, or live API keys.

Use GitHub's private vulnerability reporting flow if it is available for this repository. If not, contact the maintainer through the GitHub profile and share a minimal, redacted reproduction.

Useful details:

- AI+SU version or commit hash.
- SketchUp version and operating system.
- Provider/model name with all secrets redacted.
- Minimal prompt or generated Ruby snippet needed to reproduce the issue.
- Impact summary and whether the issue can access files, network, shell commands, or private geometry.

## Safety Boundaries

AI+SU includes basic checks before executing generated Ruby code, but those checks are not a sandbox. Users should review generated code before running it, especially code that creates, modifies, deletes, imports, exports, or communicates outside the SketchUp model.
